# PRD（产品需求文档）模板

## 与 Charter 的区别

| 维度 | Charter | PRD |
|------|---------|-----|
| 读者 | IPMT（投资决策） | 研发团队（执行） |
| 内容 | 商业价值 + 可行性 | 功能规格 + 验收标准 |
| 描述方式 | "为什么做" | "做什么"（不说"怎么做"） |
| 量化要求 | 财务数字 | 技术指标必须可测 |

## 撰写原则

1. **PRD 说 What，不说 How**——"BT 5.0 模组" 是 What；"使用 ESP32-C3" 是 How（写在技术方案里）
2. **禁止模糊词**——"快速"、"稳定"、"流畅" 必须替换为可测试的数字
3. **AC 必须可执行**——每条 AC 应能转化为测试用例
4. **第八章是硬件特有的隐形需求**——大部分软件出身的 PM 会漏掉
