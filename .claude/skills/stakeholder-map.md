# Stakeholder Map（干系人地图）

## 框架

```
                          关切度（Interest）
                      低              高
                  ┌─────────────┬─────────────┐
              高  │ 保持满意    │ 深度参与    │
                  │ (Keep       │ (Manage     │
                  │  Satisfied) │  Closely)   │
   权力             ├─────────────┼─────────────┤
   (Power)         │ 最小投入    │ 保持知情    │
              低  │ (Monitor)    │ (Keep       │
                  │              │  Informed)  │
                  └─────────────┴─────────────┘
```

## 操作步骤

### Step 1 — 列出干系人（≥ 12 人/方）

### Step 2 — 评分
对每位干系人评估：
- **权力**（1-5）：影响项目成败的能力
- **关切度**（1-5）：对本项目主动关心的程度

### Step 3 — 落入象限

### Step 4 — 制定沟通策略

| 象限 | 沟通频次 | 形式 | 内容深度 |
|------|---------|------|---------|
| 深度参与（高权高关切） | 周 | 1:1 / 评审会 | 完整数据 + 决策选项 |
| 保持满意（高权低关切） | 双周 | 简报 / 仪表盘 | 关键指标 + 例外提醒 |
| 保持知情（低权高关切） | 周 | 邮件 / 群播 | 进度 + Q&A |
| 最小投入（低权低关切） | 月 | 公开渠道 | 重大节点 |

## 智能硬件特别注意

1. **关键芯片原厂**——通常是隐藏的最高权力方，保持订单可见度比临时谈价更重要
2. **认证机构关系**——预先与实验室建立联系，预测试可省 4–8 周
3. **模具厂跑路风险**——付款节奏、模具产权归属是合同必谈项
4. **渠道商首发资源**——电商首页位置、线下端架陈列是稀缺资源，越早锁定越好
