# Charter（立项报告）模板

## 性质

Charter 是 **投资文件**，不是技术文档：
- 篇幅控制 10–15 页
- 聚焦商业价值与可行性
- 数字必须有出处（引用 CMO/CFO/CTO 报告）
- 给 IPMT 看，不是给研发团队看

## 撰写要点

1. **数字必有出处**——SAM、NPV、IRR 等关键数字标注引用源
2. **避免技术细节**——芯片型号、算法实现写在 PRD 而非 Charter
3. **风险与假设分开列**——风险是已知问题（用 pre-mortem 输出），假设是未验证前提（用 assumption-prioritization 输出）
4. **执行摘要写在最后**——但放在最前面，IPMT 通常只读这一页就做初步判断

## 联动

- 引用 `kano-model` 的产出 → 第二章关键特性
- 引用 `opportunity-solution-tree` 的产出 → 第二章产品愿景与差异化
- 引用 `pre-mortem` 的产出 → 第七章风险清单
- 引用 `assumption-prioritization` 的产出 → 第八章关键假设
- Charter 通过后由 CPO 展开为 `prd-template`
