# DCP Scoring Rubric（决策评审打分标准）

## 五维评分（每项 1-5 分）

### 1. 战略契合度（Strategic Fit）
- 5：核心战略品类，与公司 3 年路线图高度一致
- 4：相关品类，可延伸现有能力
- 3：邻接品类，需投入新能力
- 2：远离主线，需新建团队/渠道
- 1：与战略不符，纯机会主义

### 2. 市场吸引力（Market Attractiveness）
- 5：SAM ≥ 100亿，CAGR ≥ 20%，蓝海或细分蓝海
- 4：SAM 30-100亿，CAGR 10-20%
- 3：SAM 10-30亿，CAGR 5-10%
- 2：SAM < 10亿 或 红海
- 1：市场停滞或萎缩

### 3. 商业可行性（Commercial Viability）
- 5：NPV 显著为正，IRR ≥ 30%，回收期 ≤ 2 年
- 4：NPV 正，IRR 20-30%，回收期 2-3 年
- 3：NPV 略正，IRR 15-20%，回收期 3-4 年
- 2：NPV 临界，对假设高度敏感
- 1：NPV 负或回收期 > 公司上限

### 4. 执行可信度（Execution Confidence）
- 5：团队齐备 + 计划详尽 + 资源已锁定
- 4：团队齐备 + 计划清晰，资源待批
- 3：团队基本齐备，计划存在缺口
- 2：团队待组建或关键岗空缺
- 1：无明确 LPDT 或核心团队

### 5. 风险可控性（Risk Manageability）
- 5：所有高风险项都有应对方案 + 备用计划
- 4：主要风险已识别且有应对
- 3：风险已识别但应对方案不完整
- 2：存在未知重大风险
- 1：核心风险无解或未识别

---

## 通过门槛

| DCP | 核心问题 | 分数门槛 | 附加条件 |
|-----|---------|---------|---------|
| **CDCP** | 机会真实吗？值得深入吗？ | ≥ 15/25 | 任一维度不低于 2，无一票否决 |
| **PDCP** | 商业案例成立吗？团队就绪吗？ | ≥ 18/25 | Charter + PRD 完整，pre-mortem 已跑 |
| **ADCP** | 开发进展符合预期吗？市场窗口还在吗？ | ≥ 18/25 | 技术里程碑达成，BOM 偏差 ≤ 15% |
| **BDCP** | 产品质量达标吗？可量产发布吗？ | ≥ 20/25 | TR5/6 通过，认证就绪，渠道铺货计划完成 |

---

## 一票否决（直接终止，不计总分）

任一情况触发即终止项目：
- ❌ 市场规模 < 公司最低投资门槛
- ❌ 核心技术无法突破且无替代方案
- ❌ 严重违反合规/法规要求（数据隐私、出口管制）
- ❌ 投资回收期 > 公司上限（默认 4 年，按品类调整）
- ❌ 关键专利侵权风险无解
- ❌ 与公司既有产品线产生破坏性冲突且无协同方案

```
【IPMT决策 · {DCP类型}】
项目：{产品名}
日期：{YYYY-MM-DD}

━━━ 评审得分 ━━━
战略契合度：X/5 — {一句话说明}
市场吸引力：X/5 — {一句话说明}
商业可行性：X/5 — {一句话说明}
执行可信度：X/5 — {一句话说明}
风险可控性：X/5 — {一句话说明}
总分：XX/25

━━━ 一票否决检查 ━━━
[ ] 市场规模 ≥ 投资门槛
[ ] 核心技术可行
[ ] 合规无重大违规
[ ] 回收期 ≤ {X} 年
[ ] 无专利侵权
[ ] 无产品线冲突

━━━ 关键发现 ━━━
支持理由：
1. {理由}
2. {理由}

主要担忧：
1. {担忧} → 期望 PDT 在 {日期} 前回应
2. {担忧}

━━━ 决策结果 ━━━
[ ] 批准 — 授权进入下一阶段
[ ] 有条件批准 — 满足以下条件后自动通过：
    · {条件1}（截止：{日期}）
    · {条件2}
[ ] 暂缓 — 需补充：{具体内容}，{日期}重新评审
[ ] 终止 — 原因：{核心否定理由}

━━━ IPMT 经典追问（用于挑战 PDT）━━━
- 这个销量预测的依据是什么？最悲观情景呢？
- 如果核心芯片停产/涨价30%，商业案例还成立吗？
- 竞品提前3个月发布，应对策略是什么？
- 研发关键人离职风险如何缓解？
- 认证失败会推迟几个月？资金够撑吗？
```
